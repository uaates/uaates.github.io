GitHub Pages ile Site Yayınlama

HTML & CSS projeni ücretsiz olarak internete aç GitHub · Deployment · Portfolio

← Blog Ana Sayfa

GitHub Pages, HTML, CSS ve JavaScript ile hazırladığın statik web sitelerini ücretsiz olarak yayınlamanı sağlayan bir GitHub servisidir. Özellikle kişisel portföy ve blog siteleri için oldukça idealdir.

1️⃣ GitHub Pages Nedir?

GitHub Pages, GitHub üzerindeki bir repository’den otomatik olarak site oluşturan bir sistemdir. Backend gerektirmez ve tamamen ücretsizdir.

2️⃣ Repo Oluşturma

Kişisel site yayınlamak için repository adı kullaniciadi.github.io olmak zorundadır. 

uaates.github.io

Bu isimlendirme sayesinde siten doğrudan https://uaates.github.io adresinden yayınlanır.

3️⃣ Dosya Yapısı

Basit bir GitHub Pages dosya yapısı: 

/
├── index.html
├── assets/
│   ├── css/
│   │   └── main.css
│   └── images/
└── blog/

4️⃣ Git Komutları ile Yayınlama

Projeni GitHub’a göndermek için terminalden: 

git init
git add .
git commit -m "Initial site"
git branch -M main
git remote add origin https://github.com/kullaniciadi/kullaniciadi.github.io.git
git push -u origin main

5️⃣ GitHub Pages Ayarları

Repository oluşturduktan sonra:

  • Settings → Pages
  • Source: Deploy from a branch
  • Branch: main / root

Bu ayarlardan sonra GitHub Pages otomatik olarak siteyi yayınlar.

6️⃣ Avantajları

  • Ücretsiz
  • HTTPS destekli
  • Kolay deployment
  • Portföy için ideal

7️⃣ Dezavantajları

  • Backend çalışmaz
  • Sadece statik dosyalar

8️⃣ Sonuç

Eğer kişisel bir portföy, blog ya da küçük bir tanıtım sitesi yapmak istiyorsan, GitHub Pages en hızlı ve en temiz çözümlerden biridir. Özellikle yazılımcılar için CV’ye ciddi değer katar.
